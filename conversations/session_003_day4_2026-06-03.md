## Технические заметки

### NTFS Mount Truncation (критично!)
**Проблема:** `Write` tool через Cowork обрезает файлы при записи через NTFS mount.
**Симптом:** файл показывает success, но в bash только ~900 символов вместо 3000+.
**Решение:** Писать все файлы через `python3 << 'PYEOF' ... PYEOF` в bash напрямую.
**Правило:** Любой Python/PS1 файл > 50 строк — только через bash Python heredoc.

## Архитектура Knowledge Base

```
POST /analyze
  └─ MockLLMRouter.analyze()
       ├─ ScamAnalyzer (regex patterns, score)
       ├─ KnowledgeBaseClient.similarity_search(text, n=3, min=0.4)
       │    └─ ChromaDB PersistentClient
       │         └─ all-MiniLM-L6-v2 embeddings (384-dim, cosine)
       └─ AnalysisResponse(similar_cases=[...])
```

**Lazy init:** KB инициализируется при первом запросе, не при старте сервера.
**Graceful degradation:** Если chromadb недоступен → `similar_cases: []`, сервер работает.
**Cache:** `chroma_db/` папка создаётся автоматически, эмбеддинги кэшируются.
